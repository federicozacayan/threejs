Contenido cargado desde archivos externos

Si carga modelos o texturas desde archivos externos, debido a los navegadores [link:http://en.wikipedia.org/wiki/Same_origin_policy same origin policy] restricciones de seguridad, la carga desde un sistema de archivos fallará con una excepción de seguridad.

Para resolver esto, ejecute archivos desde un servidor web local. Esto le permite acceder a su página como:

http://localhost/tuArchivo.html

Si bien también es posible cambiar la configuración de seguridad del navegador en lugar de ejecutar un servidor local, no recomendamos ese enfoque. Si lo hace, puede abrir su dispositivo a vulnerabilidades, si se utiliza el mismo navegador para navegar por Internet con regularidad. El uso de un servidor local es una práctica estándar en el desarrollo web, y a continuación explicamos cómo instalar y usar un servidor local.

Ejecutar un servidor local

Muchos lenguajes de programación tienen servidores HTTP simples incorporados. No tienen tantas funciones como los servidores de producción como [link:https://www.apache.org/ Apache] o [link:https://nginx.org NGINX], sin embargo, deberían ser suficientes para probar su aplicación three.js.

Complementos para editores de código populares

Algunos editores de código tienen complementos que generarán un servidor simple a pedido.

  • [link:https://marketplace.visualstudio.com/items?itemName=yandeu.five-server Five Server] para Visual Studio Code.
  • [link:https://marketplace.visualstudio.com/items?itemName=ritwickdey.LiveServer Live Server] para Visual Studio Code.
  • [link:https://atom.io/packages/atom-live-server Live Server] para Atom.

Servez

[link:https://greggman.github.io/servez Servez] es un servidor simple con una GUI.

Node.js five-server

Servidor de desarrollo con capacidad de recarga en vivo. Instalar:

# Quitar live-server (si lo tenías) npm -g rm live-server # Install five-server npm -g i five-server # Actualizar five-server (de vez en cuando) npm -g i five-server@latest

Para ejecutar (desde la carpeta actual):

five-server . -p 8000

Node.js http-server

Node.js tiene un paquete de servidor HTTP simple. Para instalar:

npm install http-server -g

Para ejecutar (desde la carpeta actual):

http-server . -p 8000

Servidor con Python

Si tienes [link:http://python.org/ Python] instalado, debería ser suficiente ejecutar esto desde una línea de comando (desde la carpeta actual):

//Python 2.x python -m SimpleHTTPServer //Python 3.x python -m http.server

Esto servirá archivos del directorio actual en localhost bajo el puerto 8000, es decir, en la barra de direcciones escriba:

http://localhost:8000/

Servidor con Ruby

Si tiene instalado Ruby, puede obtener el mismo resultado ejecutando esto en su lugar:

ruby -r webrick -e "s = WEBrick::HTTPServer.new(:Port => 8000, :DocumentRoot => Dir.pwd); trap('INT') { s.shutdown }; s.start"

Servidor con PHP

PHP también tiene un servidor web incorporado, desde la versión php 5.4.0:

php -S localhost:8000

Lighttpd

Lighttpd es un servidor web de uso general muy ligero. Cubriremos la instalación en OSX con HomeBrew aquí. A diferencia de los otros servidores discutidos aquí, lighttpd es un servidor completo listo para producción.

  1. Instalarlo a través de homebrew brew install lighttpd
  2. Cree un archivo de configuración llamado lighttpd.conf en el directorio donde desea ejecutar su servidor web. Hay una demo [link:http://redmine.lighttpd.net/projects/lighttpd/wiki/TutorialConfiguration aqui].
  3. En el archivo conf, cambie server.document-root al directorio desde el que desea entregar los archivos.
  4. Comenzar con lighttpd -f lighttpd.conf
  5. Navegue a http://localhost:3000/ y servirá archivos estáticos desde el directorio que elija.

IIS

Si está utilizando Microsoft IIS como servidor web. Agregue una configuración de tipo MIME con respecto a la extensión .fbx antes de cargar.

File name extension: fbx MIME Type: text/plain

De forma predeterminada, IIS bloquea las descargas de archivos .fbx y .obj. Debe configurar IIS para permitir que este tipo de archivos se puedan descargar.

Otras alternativas simples son [link:http://stackoverflow.com/q/12905426/24874 tratadas aqui] en Stack Overflow.
