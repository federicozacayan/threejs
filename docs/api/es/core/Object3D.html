[name]

Esta es la clase base para la mayoría de los objetos en three.js y proporciona un conjunto de propiedades y métodos para manipular objetos en el espacio 3D.

Tenga en cuenta que esto se puede usar para agrupar objetos a través del método [page:.add]( object ) que agrega el objeto como un elemento secundario; sin embargo, es mejor usar [page:Group] para esto.

Constructor

[name]()

El constructor no toma argumentos.

Properties

[property:AnimationClip animations]

Matriz con clips de animación del objeto.

[property:Boolean castShadow]

Si el objeto se proyecta sombras. El valor predeterminado es `false`.

[property:Array children]

Matriz con los hijos del objeto. Ver [page:Group] para obtener información sobre la agrupación manual de objetos.

[property:Material customDepthMaterial]

Material de profundidad personalizado que se utilizará al renderizar en el mapa de profundidad. Solo se puede usar en el contexto de mallas. Al proyectar sombras con [page:DirectionalLight] o [page:SpotLight], si está modificando las posiciones de los vértices en el vertex shader, debe especificar un CustomDepthMaterial para las sombras adecuadas. El valor predeterminado es `undefined`.

[property:Material customDistanceMaterial]

Igual que [page:.customDepthMaterial customDepthMaterial], pero usado con [page:PointLight]. El valor predeterminado es `undefined`.

[property:Boolean frustumCulled]

Cuando se configura, verifica en cada frame si el objeto está en el frustum de la cámara antes de renderizar el objeto. Si se establece en `false`, el objeto se renderiza en cada frame incluso si no está en el frustum de la cámara. El valor predeterminado es `true`.

[property:Integer id]

Solo lectura: número único para esta instancia de objeto.

[property:Boolean isObject3D]

Solo lectura: flag para verificar si un objeto dado es del tipo [name].

[property:Layers layers]

La pertenencia a la capa del objeto. El objeto solo es visible si tiene al menos una capa en común con la [page:Camera] en uso. Esta propiedad también se puede usar para filtrar objetos no deseados en pruebas de intersección de rayos cuando se usa [page:Raycaster].

[property:Matrix4 matrix]

La matriz de transformación local.

[property:Boolean matrixAutoUpdate]

Cuando se establece, calcula la matriz de posición (rotación o cuaternión) y escala cada frame y también vuelve a calcular la propiedad matrixWorld. El valor predeterminado es [page:Object3D.DefaultMatrixAutoUpdate] (true).

[property:Matrix4 matrixWorld]

La transformación global del objeto. Si Object3D no tiene padre, entonces es idéntico a la transformación local [page:.matrix].

[property:Boolean matrixWorldAutoUpdate]

El valor predeterminado es `true`. Si se establece, el renderizador verifica cada frame si el objeto y sus elementos secundarios necesitan actualizaciones de matriz. Cuando no lo es, usted mismo debe mantener todas las matrices en el objeto y sus elementos secundarios.

[property:Boolean matrixWorldNeedsUpdate]

Cuando se establece, calcula matrixWorld en ese frame y restablece esta propiedad en `false`. El valor predeterminado es `false`.

[property:Matrix4 modelViewMatrix]

Esto se pasa al shader y se usa para calcular la posición del objeto.

[property:String name]

Nombre opcional del objeto (no es necesario que sea único). El valor predeterminado es una cadena vacía.

[property:Matrix3 normalMatrix]

Esto se pasa al shader y se usa para calcular la iluminación del objeto. Es la transposición de la inversa de la submatriz 3x3 superior izquierda de modelViewMatrix de este objeto.

La razón de esta matriz especial es que el simple uso de modelViewMatrix podría dar como resultado una longitud no unitaria de normales (en escala) o en una dirección no perpendicular (en escala no uniforme).

Por otro lado, la parte de traducción de modelViewMatrix no es relevante para el cálculo de las normales. Por lo tanto, un Matrix3 es suficiente.

[property:Function onAfterRender]

Una callback opcional que se ejecuta inmediatamente después de renderizar un objeto 3D. Esta función se llama con los siguientes parámetros: renderizador, escena, cámara, geometría, material, grupo.

Tenga en cuenta que esta callback solo se ejecuta para objetos 3D `renderizables`. Es decir, objetos 3D que definen su apariencia visual con geometrías y materiales como instancias de [page:Mesh], [page:Line], [page:Points] o [page:Sprite]. Las instancias de [page:Object3D], [page:Group] o [page:Bone] no se pueden renderizar y, por lo tanto, esta callback no se ejecuta para dichos objetos.

[property:Function onBeforeRender]

Una callback opcional que se ejecuta inmediatamente antes de que se represente un objeto 3D. Esta función se llama con los siguientes parámetros: renderizador, escena, cámara, geometría, material, grupo.

Tenga en cuenta que esta callback solo se ejecuta para objetos 3D `renderizables`. Es decir, objetos 3D que definen su apariencia visual con geometrías y materiales como instancias de [page:Mesh], [page:Line], [page:Points] o [page:Sprite]. Las instancias de [page:Object3D], [page:Group] o [page:Bone] no se pueden renderizar y, por lo tanto, esta callback no se ejecuta para dichos objetos.

[property:Object3D parent]

Padre del objeto en el [link:https://en.wikipedia.org/wiki/Scene_graph scene graph]. Un objeto puede tener como máximo un padre.

[property:Vector3 position]

Un [page:Vector3] que representa la posición local del objeto. El valor predeterminado es `(0, 0, 0)`.

[property:Quaternion quaternion]

Rotación local del objeto como [page:Quaternion Quaternion].

[property:Boolean receiveShadow]

Si el material recibe sombras. El valor predeterminado es `false`.

[property:Number renderOrder]

Este valor permite anular el orden de representación predeterminado de los objetos [link:https://en.wikipedia.org/wiki/Scene_graph scene graph] aunque los objetos opacos y transparentes permanecen ordenados de forma independiente. Cuando esta propiedad se establece para una instancia de [page:Group Group], todos los objetos descendientes se ordenarán y representarán juntos. La clasificación es de menor a mayor renderOrder. El valor predeterminado es `0`.

[property:Euler rotation]

Rotación local del objeto (ver [link:https://en.wikipedia.org/wiki/Euler_angles Euler angles]), en radianes.

[property:Vector3 scale]

La escala local del objeto. El valor predeterminado es [page:Vector3]( 1, 1, 1 ).

[property:Vector3 up]

Esto lo utiliza el método [page:.lookAt lookAt], por ejemplo, para determinar la orientación del resultado.
El valor predeterminado es [page:Object3D.DefaultUp], es decir, `( 0, 1, 0 )`.

[property:Object userData]

Un objeto que se puede utilizar para almacenar datos personalizados sobre Object3D. No debe contener referencias a funciones, ya que estas no se clonarán.

[property:String uuid]

[link:http://en.wikipedia.org/wiki/Universally_unique_identifier UUID] de esta instancia de objeto. Esto se asigna automáticamente, por lo que no debe editarse.

[property:Boolean visible]

El objeto se representa si es `true`. El valor predeterminado es `true`.

Propiedades estáticas

Las propiedades y los métodos estáticos se definen por clase en lugar de por instancia de esa clase. Esto significa que cambiar [page:Object3D.DefaultUp] o [page:Object3D.DefaultMatrixAutoUpdate] cambiará los valores de [page:.up up] y [page:.matrixAutoUpdate matrixAutoUpdate] para `cada` instancia de Object3D (o clases derivadas) ) creado después de realizar el cambio (los Object3D ya creados no se verán afectados).

[property:Vector3 DefaultUp]

La dirección predeterminada [page:.up up] para los objetos, también se usa como la posición predeterminada para [page:DirectionalLight], [page:HemisphereLight] y [page:Spotlight] (que crea luces que brillan de arriba hacia abajo).
Establecido en ( 0, 1, 0 ) de forma predeterminada.

[property:Boolean DefaultMatrixAutoUpdate]

La configuración predeterminada para [page:.matrixAutoUpdate matrixAutoUpdate] para Object3D recién creados.

Métodos

[page:EventDispatcher EventDispatcher] los métodos están disponibles en esta clase.

[method:this add]( [param:Object3D object], ... )

Agrega `objeto` como hijo de este objeto. Se puede agregar un número arbitrario de objetos. Se eliminará cualquier padre actual de un objeto pasado aquí, ya que un objeto puede tener como máximo un padre.

Consulte [page:Group] para obtener información sobre cómo agrupar objetos manualmente.

[method:undefined applyMatrix4]( [param:Matrix4 matrix] )

Aplica la transformación de matriz al objeto y actualiza la posición, la rotación y la escala del objeto.

[method:this applyQuaternion]( [param:Quaternion quaternion] )

Aplica la rotación representada por el cuaternión al objeto.

[method:this attach]( [param:Object3D object] )

Agrega `object` como hijo, mientras mantiene la transformación del mundo del objeto.

Nota: Este método no admite gráficos de escena que tengan nodos con escalas no uniformes.

[method:Object3D clone]( [param:Boolean recursive] )

recursivo: si es `true`, los descendientes del objeto también se clonan. El valor predeterminado es `true`.

Devuelve un clon de este objeto y, opcionalmente, todos los descendientes.

[method:this copy]( [param:Object3D object], [param:Boolean recursive] )

recursivo: si es `true`, los descendientes del objeto también se copian. El valor predeterminado es `true`.

Copie el objeto dado en este objeto. Nota: los detectores de eventos y las callbacks definidas por el usuario ([page:.onAfterRender] and [page:.onBeforeRender]) no se copian.

[method:Object3D getObjectById]( [param:Integer id] )

id -- Número único de la instancia del objeto

Busca a través de un objeto y sus elementos secundarios, comenzando con el objeto mismo, y devuelve el primero con una identificación coincidente.
Tenga en cuenta que los identificadores se asignan en orden cronológico: 1, 2, 3, ..., aumentando en uno por cada nuevo objeto.

[method:Object3D getObjectByName]( [param:String name] )

name: cadena para que coincida con la propiedad Object3D.name de los niños.

Busca a través de un objeto y sus elementos secundarios, comenzando con el objeto mismo, y devuelve el primero con un nombre coincidente.
Tenga en cuenta que para la mayoría de los objetos, el nombre es una cadena vacía de forma predeterminada. Vas a tiene que configurarlo manualmente para hacer uso de este método.

[method:Object3D getObjectByProperty]( [param:String name], [param:Any value] )

name -- el nombre de la propiedad a buscar.
value -- valor de la propiedad dada.

Busca a través de un objeto y sus elementos secundarios, comenzando con el objeto mismo, y devuelve el primero con una propiedad que coincide con el valor dado.

[method:Vector3 getWorldPosition]( [param:Vector3 target] )

[page:Vector3 target] — el resultado se copiará en este Vector3.

Devuelve un vector que representa la posición del objeto en el espacio mundial.

[method:Quaternion getWorldQuaternion]( [param:Quaternion target] )

[page:Quaternion target] — el resultado se copiará en este Quaternion.

Devuelve un cuaternión que representa la rotación del objeto en el espacio mundial.

[method:Vector3 getWorldScale]( [param:Vector3 target] )

[page:Vector3 target] — el resultado se copiará en este Vector3.

Devuelve un vector de los factores de escala aplicados al objeto para cada eje en el espacio mundial.

[method:Vector3 getWorldDirection]( [param:Vector3 target] )

[page:Vector3 target] — el resultado se copiará en este Vector3.

Devuelve un vector que representa la dirección del eje z positivo del objeto en el espacio mundial.

[method:Vector3 localToWorld]( [param:Vector3 vector] )

vector - Un vector que representa una posición en el espacio local de este objeto.

Convierte el vector del espacio local de este objeto al espacio mundial.

[method:undefined lookAt]( [param:Vector3 vector] )
[method:undefined lookAt]( [param:Float x], [param:Float y], [param:Float z] )

vector - Un vector que representa una posición en el espacio mundial.

Opcionalmente, los componentes [page:.x x], [page:.y y] and [page:.z z] de la posición del espacio mundial.

Gira el objeto para mirar hacia un punto en el espacio mundial.

Este método no admite objetos que tengan elementos primarios no uniformemente escalados.

[method:Array raycast]( [param:Raycaster raycaster], [param:Array intersects] )

Método abstracto (vacío) para obtener intersecciones entre un rayo emitido y este objeto. Las subclases como [page:Mesh], [page:Line] y [page:Points] implementan este método en orden para usar la transmisión de rayos.

[method:this remove]( [param:Object3D object], ... )

Elimina `object` como hijo de este objeto. Se puede eliminar un número arbitrario de objetos.

[method:this removeFromParent]()

Elimina este objeto de su padre actual.

[method:this clear]()

Elimina todos los objetos hijos.

[method:this rotateOnAxis]( [param:Vector3 axis], [param:Float angle] )

axis - un vector normalizado en el espacio de objetos.
angle - El ángulo en radianes.

Girar un objeto a lo largo de un eje en el espacio de objetos. Se supone que el eje está normalizado.

[method:this rotateOnWorldAxis]( [param:Vector3 axis], [param:Float angle] )

axis - un vector normalizado en el espacio de objetos.
angle - El ángulo en radianes.

Girar un objeto a lo largo de un eje en el espacio mundial. Se supone que el eje está normalizado. Método Supone que no hay padre rotado.

[method:this rotateX]( [param:Float rad] )

rad - el ángulo de rotación en radianes.

Gira el objeto alrededor del eje x en el espacio local.

[method:this rotateY]( [param:Float rad] )

rad - el ángulo a girar en radianes.

Gira el objeto alrededor del eje y en el espacio local.

[method:this rotateZ]( [param:Float rad] )

rad - el ángulo de rotación en radianes.

Gira el objeto alrededor del eje z en el espacio local.

[method:undefined setRotationFromAxisAngle]( [param:Vector3 axis], [param:Float angle] )

axis - un vector normalizado en el espacio de objetos.
angle - El ángulo en radianes.

Llama [page:Quaternion.setFromAxisAngle setFromAxisAngle]( [page:Float axis], [page:Float angle] ) en la [page:.quaternion].

[method:undefined setRotationFromEuler]( [param:Euler euler] )

euler - Ángulo de Euler que especifica la cantidad de rotación.

Llama [page:Quaternion.setRotationFromEuler setRotationFromEuler]( [page:Euler euler]) en la [page:.quaternion].

[method:undefined setRotationFromMatrix]( [param:Matrix4 m] )

m - rotar el cuaternión por el componente de rotación de la matriz.
Llama [page:Quaternion.setFromRotationMatrix setFromRotationMatrix]( [page:Matrix4 m]) en la [page:.quaternion].

Tenga en cuenta que esto supone que el 3x3 superior de m es una matriz de rotación pura (es decir, sin escala).

[method:undefined setRotationFromQuaternion]( [param:Quaternion q] )

q - Quaternion normalizado.

Copia el quaternion dado en [page:.quaternion].

[method:Object toJSON]( [param:Object meta] )

meta - objeto que contiene metadatos como materiales, texturas o imágenes para el objeto.
Convierte el objeto a three.js [link:https://github.com/mrdoob/three.js/wiki/JSON-Object-Scene-format-4 JSON Object/Scene format].

[method:this translateOnAxis]( [param:Vector3 axis], [param:Float distance] )

axis - un vector normalizado en el espacio de objetos. distance - La distancia a traducir. Traslada un objeto por distancia a lo largo de un eje en el espacio de objetos. Se supone que el eje está normalizado.

[method:this translateX]( [param:Float distance] )

Traduce el objeto a lo largo del eje x en el espacio del objeto en unidades de `distance`.

[method:this translateY]( [param:Float distance] )

Traduce el objeto a lo largo del eje y en el espacio del objeto en unidades de `distance`.

[method:this translateZ]( [param:Float distance] )

Traduce el objeto a lo largo del eje z en el espacio del objeto en unidades de `distance`.

[method:undefined traverse]( [param:Function callback] )

callback - Una función que tiene como primer argumento un objeto object3D.

Ejecuta la callback en este objeto y todos sus descendientes.
Nota: No se recomienda modificar el gráfico de escena dentro de la callback.

[method:undefined traverseVisible]( [param:Function callback] )

callback - Una función que tiene como primer argumento un objeto object3D.

Como traverse, pero la callback solo se ejecutará para objetos visibles. Los descendientes de objetos invisibles no se recorren.
Nota: No se recomienda modificar el gráfico de escena dentro de la callback.

[method:undefined traverseAncestors]( [param:Function callback] )

callback - Una función que tiene como primer argumento un objeto object3D.

Ejecuta la callback en todos los antepasados.
Nota: No se recomienda modificar el gráfico de escena dentro de la callback.

[method:undefined updateMatrix]()

Actualiza la transformación local.

[method:undefined updateMatrixWorld]( [param:Boolean force] )

force - Un valor booleano que se puede usar para omitir [page:.matrixWorldAutoUpdate], para volver a calcular la matriz global del objeto y los descendientes en el frame actual. Útil si no puede esperar a que el renderizador lo actualice en el siguiente frame (suponiendo que [page:.matrixWorldAutoUpdate] es `true`).

Actualiza la transformación global del objeto y sus descendientes si la matriz global necesita actualización ([page:.matrixWorldNeedsUpdate] es true) o si el parámetro `force` es `true`.

[method:undefined updateWorldMatrix]( [param:Boolean updateParents], [param:Boolean updateChildren] )

updateParents - actualiza recursivamente la transformación global de los ancestros.
updateChildren - actualiza recursivamente la transformación global de los descendientes.

Actualiza la transformación global del objeto.

[method:Vector3 worldToLocal]( [param:Vector3 vector] )

vector -Un vector que representa una posición en el espacio mundial.

Convierte el vector del espacio global al espacio local de este objeto.
